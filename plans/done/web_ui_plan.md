# Plan: Web UI for GoGame

## Context
The GoGame currently has a CLI-only interface (script.ts with ANSI colors and Unicode stones). Adding a Web UI will make the game more accessible — users can play in a browser on desktop or mobile.

## File Structure

```
src/web/
  index.html    (single-page shell)
  style.css     (layout, board grid, stones, responsive)
  app.ts        (entry point — state, rendering, event handlers)
  bundle.js     (generated by bun build, .gitignore)
package.json    (already exists, add build script)
```

## Build Pipeline

```bash
bun build src/web/app.ts --outfile=src/web/bundle.js --target=browser
```

Add to package.json scripts: "build:web": "bun build src/web/app.ts --outfile=src/web/bundle.js --target=browser"

## Files to Create/Modify

### 1. src/web/index.html

Single-page shell with:
- link rel="stylesheet" href="style.css"
- A div#app container
- script src="bundle.js" at end of body
- Meta viewport tag for mobile

Layout structure (generated by JS, but template in HTML):
- Top bar: game title, controls (New Game, Pass, Resign, Help)
- Board area: centered, responsive
- Status bar: current player, captures, move count, last move
- Side panel (desktop) / bottom sheet (mobile): controls + game info

### 2. src/web/style.css

Layout:
- Two-column on desktop (>=768px): side panel (320px) + board (flex:1)
- Single-column on mobile (<768px): board then controls below
- Board centered within its container

CSS Grid Board:
- N x N grid of .intersection cells
- Each cell is a square (aspect-ratio: 1)
- Grid lines via ::before and ::after pseudo-elements on each cell:
  - ::before = horizontal line (full width, 1px height, centered vertically)
  - ::after = vertical line (full height, 1px width, centered horizontally)
- Lines truncated at board edges via classes: .edge-top, .edge-bottom, .edge-left, .edge-right
- Star points: radial-gradient circle at center of cell
- Cell size: 32px on desktop, ~8vw on mobile (min 24px, max 36px)

Stones:
- 90% of cell size, border-radius: 50%, positioned absolute centered
- Black stone: radial-gradient(circle at 35% 35%, #555, #000)
- White stone: radial-gradient(circle at 35% 35%, #fff, #ccc) with box-shadow for depth
- Placed via JS (createElement, classList.add)
- Last-move marker: small red circle via ::after (6px, bottom-right)
- Hover ghost: semi-transparent stone preview on empty intersections

Responsive:
- Media query at 768px switches from 2-col to single-col
- Board scales to fit viewport width with max-width constraint
- Controls as flex-wrap row below board on mobile

### 3. src/web/app.ts

Imports from engine.ts:
- createInitialState, placeStone, pass, resign, countScore, isValidMove
- BLACK, WHITE, EMPTY
- type GameState

Imports from bots.ts:
- createRandomBot, createGreedyBot, createHeuristicBot
- Bot interface / BotFactory

State:
```
const state = {
  game: createInitialState(9),
  bots: { black: null, white: null },  // null = human
  selectedLevel: 3,
  playerColor: BLACK,
}
```

Functions:

- renderBoard():
  - Build N x N grid of .intersection divs
  - Each div has data-r and data-c attributes
  - Apply edge classes for grid line truncation
  - Place stone divs for non-empty cells
  - Add ghost preview on hover
  - Mark last move with red dot

- renderStatus():
  - Update status bar: turn count, current player, captures, consecutive passes, last move

- handleClick(r, c):
  - If gameOver, ignore
  - If bot owns current turn, ignore
  - Call placeStone(state.game, r, c)
  - If invalid, shake/feedback
  - Re-render board + status
  - If !gameOver && next player is a bot, schedule botMove()

- botMove():
  - Determine which bot to use based on currentPlayer
  - Call bot.selectMove(state.game, currentPlayer)
  - If null, call pass(state.game)
  - Else call placeStone(state.game, move.r, move.c)
  - Re-render
  - If !gameOver && next is bot, schedule next botMove with setTimeout 300ms

- newGame():
  - Reset state.game = createInitialState(selectedSize)
  - Reset bot assignments
  - Re-render

- init():
  - Read query params for size, level, color (optional)
  - Create initial state
  - Build DOM structure
  - Render
  - If first player is bot, start bot chain

AI Integration:
- Difficulty dropdown (1=Random, 2=Greedy, 3=Heuristic)
- Color picker (play as Black or White)
- Both can be changed mid-game (resets)
- Bot plays automatically with 300ms delay for visual pacing
